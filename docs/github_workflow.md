# GitHub Workflow

This project should use small, meaningful Git commits and clear release tags.

## Commit Rules

- Commit after each completed sub-phase.
- Keep commits focused on one kind of work.
- Use clear messages such as `Phase 0.3: add initial documentation`.
- Update `docs/project_memory.md` before committing.

## Push Rules

- Push after meaningful milestones.
- Push after phase completions.
- Do not push incomplete implementation work unless it is intentionally saved as a work-in-progress branch.

## Tag Rules

- Tag documented milestones and releases.
- Suggested tags:
  - `v0.1.0-docs` after Phase 1.
  - `v0.2.0-backend-mvp` after Phase 2.
  - `v0.3.0-frontend-mvp` after Phase 3.
  - `v0.4.0-examples-tests` after Phase 4.
  - `v1.0.0-mvp1` after Phase 5.

## Beginner-Friendly Workflow

Recommended loop:

1. Read `docs/project_memory.md`.
2. Work on the current sub-phase only.
3. Update `docs/project_memory.md`.
4. Review changed files.
5. Commit the sub-phase.
6. Push when a meaningful milestone is ready.
